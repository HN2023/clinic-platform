Architecture

Five layers, clearly separated.

Each layer has a single responsibility and an enforced security boundary. The database namespace is unreachable from the frontend — not just by convention, but by Kubernetes network policy.

01 Client layer Patient portal, staff dashboard, mobile app, admin panel React
02 API gateway Auth, routing, rate limiting — single entry point Kong / AWS API GW
03 Core services Scheduling · EHR/EMR · Billing · Notifications Node.js microservices
04 Data & integrations RDS · Redis · S3 · Labs (HL7/FHIR) · Payments AWS managed services
05 Infrastructure Kubernetes · Vault · Grafana · CI/CD pipeline EKS · ArgoCD
Deployment

12-stage CI pipeline.
Zero-downtime releases.

Every pull request runs the full pipeline — code quality, security scanning, HIPAA compliance gates, and image signing — before anything reaches staging. Production uses blue/green switching for instant rollback.

CI pipeline

Automated compliance gates

SAST scanning via Snyk, container image scanning via Trivy, and HIPAA policy checks run on every PR. Nothing merges unless all 12 stages pass.

CD pipeline

GitOps via ArgoCD

Every production change is a git commit. ArgoCD watches for new image tags and applies them automatically. No manual kubectl in production.
